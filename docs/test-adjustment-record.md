## 2. Feed / OPML / 订阅导入

### 测试关注点与调整

| 功能点 | 修改内容 |
|--------|----------|
| 添加 Feed | 测试 RSS / Atom / JSON Feed URL 的拉取、解析和入库 |
| Feed 自动发现 | 发现部分网页 URL 不能直接导入后，推动真实订阅地址自动发现 |
| OPML 导入 | 测试批量导入订阅源，并检查成功和失败明细 |
| 去重保存 | 测试重复刷新和重复导入，推动文章去重调整 |
| 错误诊断 | 针对网络失败、请求超时、无法解析等情况补充错误反馈 |
| Feed 删除 | 从测试使用角度补充右键删除订阅源的管理能力 |

### 问题处理

| 问题 | 处理方式 |
|------|----------|
| 部分 URL 不是合法 Feed | 增加导入诊断与自动发现逻辑 |
| 重复刷新导致文章重复 | 使用数据库唯一约束和保存逻辑去重 |
| 订阅源较多时导入反馈不清晰 | 通过导入结果报告展示成功数量和失败列表 |
| Feed 列表操作不方便 | 增加右键删除等管理能力 |

## 3. 阅读器界面与交互测试

### 测试关注点与调整

| 区域 | 修改内容 |
|------|----------|
| 三栏布局 | 检查左侧订阅源、中间文章列表、右侧阅读器的主流程是否顺畅 |

- 搜索、列宽、滚动等日常操作更符合桌面应用习惯。

## 4. 正文清洗与文章内容

### 测试关注点与调整

| 功能点 | 修改内容 |
|--------|----------|
| 正文清洗 | 测试 `raw_html -> cleaned_html -> cleaned_markdown` 管线，反馈清洗展示问题 |
| 打包环境 | 发现 release 构建中 cleaner 不稳定后推动修复 |
| 失败兜底 | 反馈 Node cleaner 管道提前关闭导致的正文失败问题 |
| 内容展示 | 发现部分文章正文渲染异常、Feed 内容与清洗内容不一致的问题 |

### 处理过的典型现象

- 清洗后正文没有显示；
- release 包中 cleaner 运行失败；
- `pipe closed` 导致文章内容无法展示；
- 内链、PDF、复杂页面导致正文预览不稳定。

## 5. LLM、翻译与摘要

### 测试关注点与调整

| 功能点 | 修改内容 |
|--------|----------|
| Provider 设置 | 测试 LLM Provider 配置、API 地址、模型、语言选项的可用性 |
| 翻译 | 反馈翻译语言、段落对齐、选中文本翻译等问题 |
| 摘要 | 测试摘要语言设置、摘要面板空状态和开关行为 |
| 诊断 | 反馈 LLM 请求失败、代理和接口返回异常时缺少排查信息 |
| 隐私 | 发现 seeded API key 风险后推动移除默认敏感配置 |

### 隐私边界

相关调整遵守项目隐私规则：文章内容只在用户主动点击摘要或翻译时发送给用户配置的 LLM Provider；业务代码不绑定单一模型厂商，API key 不提交到仓库。

## 9. 当前验收建议

如需重新验收本文涉及的主线功能，建议执行：

```bash
cd app
npm run build

cd src-tauri
cargo check
```

手动验收建议：

1. 启动 `npm run tauri dev`，确认应用窗口正常打开；
2. 添加一个 RSS Feed，刷新 2-3 次，确认文章不重复异常增长；
3. 导入 OPML，确认成功和失败数量有明确反馈；
4. 打开文章，确认正文、摘要、翻译、查看原文入口可用；
5. 配置 LLM Provider，确认 API key 只保存在本地凭据存储；
6. 删除 Feed、切换 Favorites / Read Later / All 视图，确认列表状态正确。
